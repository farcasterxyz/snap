# Agents

This page collects everything an AI agent or automated tool needs to work with Farcaster
Snaps.

## Machine-readable documentation

The full docs are available in machine-readable form:

- **[/llms.txt](/llms.txt)** — all documentation concatenated into a single plain-text
  file, suitable for pasting into a context window or fetching at the start of a session
- **`/markdown-content/<slug>`** — individual pages as plain markdown (e.g.
  `/markdown-content/elements`, `/markdown-content/buttons`)

## Creating a snap with an agent

The fastest path is the Claude Code skill:

```bash
mkdir -p $HOME/.claude/skills/create-farcaster-snap
curl -sS -o $HOME/.claude/skills/create-farcaster-snap/SKILL.md https://docs.farcaster.xyz/snap/SKILL.md
```

Then run:

```
/create-farcaster-snap a poll asking users to pick their favorite L2
```

The skill reads the spec, generates valid snap JSON, validates it, and deploys to a live
URL. It is the source of truth for the create-snap workflow — fetch the latest version
at the URL above rather than relying on a cached copy.

## Key facts for agents

- **Snap GET**: send `Accept: application/vnd.farcaster.snap+json`; the server returns
  snap JSON or falls back to HTML
- **Snap POST**: body must be a JFS compact string (`header.payload.signature`); skip
  verification locally with `SKIP_JFS_VERIFICATION=1`
- **Validator package**: `@farcaster/snap` (Zod schemas + runtime validation). Run
  `pnpm --filter @farcaster/snap test` to check your output
- **Template**: [`template/`](https://github.com/farcasterxyz/snap/tree/main/template)
  is the canonical deployable starter — use it rather than building from scratch
- **Deploy target**: [host.neynar.app](https://host.neynar.app) — fetch `/SKILL.md`
  there for the latest deploy instructions
